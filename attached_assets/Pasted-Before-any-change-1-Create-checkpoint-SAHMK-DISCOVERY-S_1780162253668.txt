====================================================
OBJECTIVE

Extend SAHMK Discovery Console into a lightweight Saudi Market Data Discovery and Storage layer.

The purpose is to:

* Discover exactly which SAHMK datasets are available under the current subscription.
* Download available datasets on demand.
* Persist downloaded datasets locally.
* Keep a small rolling history.
* Allow future modules (Thesis Engine, Research Engine, Scenario Engine, etc.) to consume this layer later.

This layer must remain completely independent from Holdings and Portfolio Valuation.

====================================================
UI LAYOUT

Reorder the SAHMK Discovery tab as follows:

1. SAHMK Discovery Console header
2. Saudi symbol input
3. Action buttons:

* Run Discovery
* Download & Store Available Data
* Load Stored Data

4. Stored SAHMK Data

(MUST appear immediately after buttons)

5. Latest Discovery Summary
6. Endpoint Detail

The user must never need to scroll through endpoint results to see stored data.

====================================================
STORED SAHMK DATA SECTION

Add section:

“Stored SAHMK Data”

Display:

* Symbol
* Dataset name
* Last fetched timestamp
* Record count
* Available stored versions
* File name
* Download JSON
* View / Load

If no stored data exists:

“No stored SAHMK data found for this symbol.”

====================================================
AVAILABLE DATASETS

Store only datasets that successfully return data.

Examples currently observed:

* Quote
* Historical Prices
* Market Summary

Do NOT store datasets returning HTTP 404 or empty payloads.

====================================================
LOCAL STORAGE

Storage structure:

data/sahmk_discovery/{symbol}/{dataset}/

Examples:

data/sahmk_discovery/6004/quote/
data/sahmk_discovery/6004/historical_prices/
data/sahmk_discovery/6004/market_summary/
data/sahmk_discovery/6004/discovery_report/

File naming:

{dataset}_{YYYYMMDD_HHMMSS}.json

Examples:

quote_20260530_173000.json
quote_20260530_181500.json

====================================================
FILE CONTENT

Each stored file must contain:

* symbol
* source = SAHMK
* endpoint
* fetched_at
* dataset
* status
* available_fields
* raw_response

====================================================
PERSISTENCE

Stored files must survive:

* Page refresh
* Browser refresh
* App restart
* Leaving and returning to the tab

Do NOT clear stored files automatically.

====================================================
FIFO RETENTION

For each symbol and dataset type:

Keep only the latest 3 versions.

Examples:

6004 / quote:

* keep newest 3
* delete older versions automatically

6004 / historical_prices:

* keep newest 3
* delete older versions automatically

6004 / market_summary:

* keep newest 3
* delete older versions automatically

6004 / discovery_report:

* keep newest 3
* delete older versions automatically

Retention is per symbol and per dataset.

Example:

6004 quote → 3 files
2222 quote → 3 files

independently maintained.
