PORTFOLIO APP FIXES REQUIRED

1) SELL / CLOSE POSITION BUG

The Sell form is broken.

Current issue:
The system shows:
"Cannot sell xxxx shares... only 0 available"

even when the holding clearly exists in the Holdings table.

Required fix:
- Holdings and transactions must use the same normalized ticker key.
- Normalize Saudi tickers internally to .SR.
- Prevent mismatches such as:
  2222.SE
  2222.SR

These must never create separate positions.

- Sell validation must calculate available shares from actual holdings.
- Close Full Position must use the exact available quantity from holdings.

Test case:
Holding:
Ticker = 2222.SR
Qty = 7106
Avg Cost = 24.9008

Selling 2 shares must succeed.

Close Full Position must sell all 7106 shares successfully.

--------------------------------------------------

2) SELL TRANSACTION MODES

Support two modes:

A) Actual Sell
- Create sell transaction
- Reduce holding quantity
- Credit cash account
- Apply fees
- Update realized P&L

B) Record Correction Only
- Create correction transaction
- Reduce holding quantity
- DO NOT credit cash
- DO NOT affect account cash balance
- Update holdings only

--------------------------------------------------

3) BUY CASH VALIDATION

Actual Buy:
- Must verify available cash balance.
- Reject purchase if cash is insufficient.

Record Correction Buy:
- Allow purchase without cash.
- No cash movement.
- Holdings update only.

Logic:

If Record Correction = FALSE
    Check available cash
Else
    Skip cash validation
