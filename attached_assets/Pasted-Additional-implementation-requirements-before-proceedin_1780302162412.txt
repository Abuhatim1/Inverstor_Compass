Additional implementation requirements before proceeding:

Before implementing, please explicitly confirm the following points.

1. Purge stale child filter state before rendering widgets

When Market preset changes between:

* Saudi
* US
* All

the child filter session state must be validated before rendering the multiselect widgets.

Child filter keys:

* alloc_ms_sector
* alloc_ms_ccy
* alloc_ms_asset

If any stored selected value is no longer available within the new market-scoped option list, remove that session state key before rendering the widget.

This purge must happen before the multiselect widgets are created, not after.

Expected behavior:

* If user switches from US to Saudi, any previously selected US-only asset is cleared automatically.
* If user switches from Saudi to US, any previously selected Saudi-only asset is cleared automatically.
* If user switches to All, valid selections may remain only if still valid under All.
* No stale asset, sector, or currency selection should survive when it is outside the current market scope.

2. Avoid false “No holdings match” message

After changing Market preset, the app should not show:
“No holdings match the selected filters”
only because an old child filter selection remained in session state.

The correct behavior is:

* purge stale child filters first
* then render scoped filter options
* then apply filters
* then show results

3. Reset behavior must remain child-only

The Reset filters button should clear only:

* alloc_ms_sector
* alloc_ms_ccy
* alloc_ms_asset

It must not change:

* Saudi / US / All radio preset
* base currency
* valuation data
* holdings data

4. Market preset remains the parent scope

Market preset is the only market selector.

The hidden/removed Market multiselect must not be reintroduced.
